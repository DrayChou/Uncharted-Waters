# 剧情结构研究摘录

> 本文根据 `sea2_demo` 的 `README.md`、`docs/游戏逻辑说明.md` 与 `docs/PHASE1_DATA_TABLES.md` 重新整理，只摘出对本仓库最有用的部分，方便做剧情百科、角色关系整理和资料索引。

## 一句话概括

`sea2_demo` 对《大航海时代 II》的判断是：**它不是单线剧情游戏，而是“主人公身份 + 地点 + 时间 + 任务状态 + 资金 / 船只 / 旗标”共同驱动的分支网络。**

## 当前最有价值的已确认信息

按该项目文档，已经明确或较稳定的量化信息包括：

- **6 条主人公主线**
- **192 个角色名**（`Name.tbl`）
- **1022 条通用 UI / 港口对话**（`Message.dat`）
- **61 条菜单 / 命令文本**（`Menu.dat`）
- **约 5800 条已恢复文本**
- **48 个 chunk / 186 个 subscript / 745 条 dispatch edge** 的静态脚本结构
- João 开场局部已整理出 **87 个 timeline item / 142 个节点 / 208 条边**

这些数字的意义在于：二代的剧情体量并不只是“六个剧本换皮”，而是确实有一套复杂的事件容器、派发表和跨线复用结构。

## 六条主线的核心气质

按 `sea2_demo` 的整理，六位主人公大致可归纳为：

1. **葡萄牙贵族少年线**：被逐出家门，筹资造船，围绕约翰传说、王室阴谋与远航展开。
2. **西班牙军官复仇线**：追查亲人 / 恋人失踪真相，逐步从体制内调查滑向海上复仇。
3. **英格兰王命海军线**：围绕王命、私掠许可、组舰队与对抗西班牙势力展开。
4. **地理学家制图线**：以梅尔卡特和世界地图为中心，把航海、测绘与地域探索串起来。
5. **冒险 / 资助 / 寻宝线**：从资助、债务与冒险出发，逐步转向黄金之国、圣物与复兴叙事。
6. **贸易 / 发家 / 寻妹线**：从穷商人起步，经由投资、贸易、港口经营一路推到家庭与同盟任务。

## 跨线最关键的人物节点

该项目特别强调，六条线真正有意思的地方不在“分别讲了什么”，而在“哪些人物把不同主角线接起来”。较核心的交叉点包括：

- **约翰**：最核心的跨线角色
- **公爵夫人**：把葡萄牙贵族线与资助 / 冒险线连接起来
- **卡特琳娜**：连接葡萄牙线与西班牙复仇线
- **梅尔卡特**：连接制图线与冒险线
- **夏洛克 / 皮耶德**：连接冒险线与贸易线
- **蒂迪亚 / 萨达姆**：把贸易线中的爱情、伙伴与任务关系固定下来

如果后面要做人物索引、剧情百科或“谁和谁有关系”的资料图，这一层会比单纯摘录对白更有价值。

## 对本仓库最有帮助的几个结论

### 1. 二代很适合按“人物 / 地点 / 事件”三条线重新整理

本仓库已有：

- 港口布局图
- 船只图片
- 人物 / 港口 / 船只资料 PDF

而 `sea2_demo` 提供的补充视角是：

- 角色关系不是孤立的
- 港口不是单纯地理点，而是剧情触发节点
- 六条主线之间存在大量“借人、借地点、借目标”的交叉

也就是说，后续若继续扩资料，最值得做的不是单加更多零散攻略，而是把 **人物、港口、剧情触发** 三者串起来。

### 2. 港口与世界地图可做更结构化索引

该项目确认了：

- 港口相关数据是高度结构化的
- 100 个常规港口可以稳定映射到经纬度、区域和投影坐标
- 世界海域还存在与航海机制直接相关的 **30×45 风向 / 洋流网格**

这说明本仓库未来若扩成更强的资料库，完全可以继续往“地图索引 / 航路 / 海域机制”方向发展。

### 3. 不宜把 demo 派生数据直接当原版资料

`sea2_demo` 里有一些数据是为了 Godot demo 服务的，例如：

- 商品价格表
- 起始资金
- demo 港口交易配置
- shipyard 可买船型

这些很有参考价值，但未必等于原版 PC 或 MD 的权威平衡数据，所以这里只保留其结构分析结论，不把这些数值直接收入主资料。
