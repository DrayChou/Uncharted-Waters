# 大航海时代 II 复刻系统蓝图

> 这不是开发排期，而是一份把现有资料、逆向数据和未来实现层串起来的结构图。目标是：当你真正开做复刻时，能知道**先做什么、哪些数据能直接喂、哪些地方必须自己补设计**。

## 1. 复刻目标分层

建议把复刻拆成 4 层，而不是一上来照着原作把所有系统一起重做。

### L1 世界层
负责“这个世界存在什么、都在哪”。

包含：
- 世界地图底图
- 100 港坐标

- 海域与陆地边界

- 港口商品差异

## 2. 建议的技术架构

如果以后真做复刻，建议数据层不要直接和表现层绑死。

### 数据分层建议

#### A. source 层
保留原始导入，不改字段名。

例如：
- `imported/sea2_demo/game_data/*.json`
- 原始攻略表
- 原始图片资源

用途：
- 回溯来源
- 复查误差
- 避免“整理后不可逆”

#### B. normalized 层
把原始数据规范成你自己的复刻格式。

例如：
- `ports_for_remake.json`
- `wind_grid.json`
- `ship_specs.json`
- `event_nodes.json`

要求：
- 字段统一命名
- 坐标系统统一
- 不混入表现层资源路径以外的 UI 逻辑

#### C. gameplay 层
真正供游戏读取的数据。

例如：
- 开局主角模板
- 港口商店库存
- 事件触发条件
- 海战参数

这层可以有设计取舍，不必完全等于原始数据，但必须能追溯到 normalized 或 source。

## 3. 推荐的首个可玩 MVP

如果要控制范围，我建议复刻版第一阶段不要试图做完整六主角，而是做：

### MVP-1：自由航海沙盒
- 世界地图
- 风向 / 洋流
- 20~30 个核心港口
- 基础买卖系统
- 基础补给与船只状态
- 海战最小闭环

目的：先验证“航海手感”和“港口循环”对不对。

### MVP-2：单主角主线
- 选一条最适合拆的主线
- 做状态机、事件触发、人物关系
- 验证剧情系统的可维护性

推荐优先：
- **地理学家制图线** 或 **贸易 / 发家线**
- 因为这两条线对“港口 + 世界地图 + 数据结构”依赖最直观

### MVP-3：六主角统一框架
- 主角模板化
- 共用系统 + 差异化剧情
- 跨线人物与共享地点

## 4. 当前资料最值得继续深挖的三个方向

### 方向 1：给 `za_dat` 命名字段
这会直接决定：
- 港口商品系统
- 投资系统
- 发展等级
- 船厂差异

这是最值得优先继续提炼的一块。

### 方向 2：建立“港口功能点”表
把现有 `港口布局图` 与城市功能做结构化对应：
- 王宫
- 酒吧
- 商店
- 码头
- 旅馆
- 行会 / 教会 / 造船厂

这会直接帮助地图点击交互和城市场景搭建。

### 方向 3：把剧情摘录升级成事件蓝图
把现在的“六条线概述”继续细化成：
- 角色
- 地点
- 前置条件
- 触发事件
- 状态变化
- 奖励 / 解锁

这会是以后做剧情编辑器或事件 DSL 的基础。

## 5. 现阶段不要过早做的事

- 不要先做完整 UI 复刻
- 不要先追求 100% 原版数值一致
- 不要一开始就把六主角全接进来
- 不要把 source 层和 gameplay 层混成一个文件
- 不要直接把低置信度 `ships.json` / `characters.json` 当正式主数据

## 6. 一句话结论

现在这批资料已经足够支撑一个 **“世界地图 + 港口 + 风向洋流 + 航海循环”** 的复刻骨架；真正的下一道门槛，不是资源，而是：

1. 给港口数值表命名字段
2. 把港口功能结构化
3. 把六主角剧情整理成可执行的状态机模型
